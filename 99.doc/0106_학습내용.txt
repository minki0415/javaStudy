1. 4일차 학습내용

	1) 배열 / 반복문 / 조건문 syntax
	2) MVC
		- 여러명의 고객 정보를 한 번에 반환하는 기능으로 확장
	3) 상속 & 다형성
	
2. 3일차까지의 학습내용
	1) MVC pattern
		 - Model : business/biz/core/핵심 로직
		 		   java/python
		 		   
		 - View : 화면 처리 로직
		 		  html/css/java script
		 		  jsp(자바코드)
		 		  
		 - Controller : 중개, 중재, 조절
		 				java 기반의 servlet
		 				
	2) DTO pattern
		DataTransferObject
		멤버변수 / 기본생성자 / [argument]생성자 / getXxx(),setXxx() / toString() 
		
	3) ...
	
3. builder pattern 학습의 필요성에 대한 사고하는 힘 기르기
	하나의 정보가 다수의 데이터를 보유하고 있다 가정
	가령 객체 생성 즉 데이터를 사용 가능하게 생성함을 의미
	하나의 객체 생성시에 모든 보유한 멤버 변수 값을 반드시 다 초기화는 의무가 아님
	상황에 따른 초기화 안하는 변수들도 다수 존재 가능 
	
	가령
		10개의 변수를 보유한 DTO 클래스 있다 가정
		어떤 때는 7개의 데이터(변수)로 하나의 DTO
		어떤 때는 5개의 데이터(변수)로 하나의 DTO
		어떤 때는 7개의 데이터(변수)로 하나의 DTO 생성시에 기존 7개의 데이터가 아닌 다른 데이터로 생성
	DTO 클래스 개발 및 객체 생성 문법 고려?
	
	질문
		개발방법 1 : 기본 생성자로 객체 생성 후에 setXxx() 호출
		개발방법 2 : 다양한 경우의 수를 고려한 parameter가 다양한 생성자들 다수 개발해서 호출
	  O 개발방법 3 : 객체생성시 선별해서 필요한 멤버변수값만 초기화 할 수 있는 방법 고민
	  
4. 배열 
	1) 다량의 데이터를 하나의 변수로 관리하는 구조
		- 개별 데이터 구조 분리는 고유한 index 값 활용
		- 0 부터 시작
			: 문자열의 각 글자 의미하는 index도 0부터 시작
			
	2) 자바에서의 제약 조건 
		- 동일한 타입에 한해서만 배열에 저장 및 활용
	3) 기본 타입과 참조 타입으로 배열 생성 및 활용
	4) 배열 타입은 참조 타입 
	5) 배열 저장 영역은 heap
	6) 문법
		(1) [] 배열 의미하는 표현 
			- 선언, 생성, 초기화 동시에 진행
				- 타입 [] 변수 = {값1, 값2, ...}
				- new 없이도 객체 생성(String과 배열만 해당)
			
			- 선언, 생성, 초기화 분리해서 개발
				- 선언 : 타입[] 변수명 ;
				- 생성 : 변수명 = new 타입[메모리수];
				- 초기화 : 변수명[0] = 값; 
		(2) [] = 1차원 / [][] = 2차원 / [][][] = 3차원
		
	7) lenth
		- 배열이 메모리에 생성시 자동 생성되는 배열만의 변수
		- 생성된 배열의 메모리 개수
		
5. 리뷰 
	데이터 타입 종류
		기본 - byte/short/int/long | float | double | char | boolean
			- 값만 표기에 맞게 변수에 대입하여 활용
		
		참조 - 객체타입
			- API제공하는 모든 클래스 타입 / 사용자가 개발하는 클래스 타입/ 배열
 			- 객체 생성(new 생성자([..])해서 변수에 대입해서 활용 // String은 new 없이도 생성 가능, " "
 			- 객체가 보유한 변수와 메소드 호출로 활용 
 			
 	데이터 타입 사용 위치
 		변수타입(멤버 변수, 로컬 변수) / 메소드 반환 타입
